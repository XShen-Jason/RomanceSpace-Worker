💡 子域名小项目平台指南

1️⃣ 架构总览

平台整体采用 边缘 + 中心服务器 架构：

  • Cloudflare（DNS + Worker + KV + Cache）
    • 处理用户访问请求
    • 解析子域名
    • 分发请求到对应小项目模块
    • 缓存热门页面，加速响应
  • 中心服务器（香港节点推荐）
    • 用户注册、登录
    • 用户项目创建与管理
    • 数据持久化（数据库）
    • 同步子域名映射到 CF KV
    • 后台管理与统计
原则：读操作在边缘完成，写操作在服务器完成。

2️⃣ 服务器端职责

  • 用户系统
    • 注册、登录、权限管理
    • 用户信息存储
  • 小项目管理
    • 用户选择或创建小项目
    • 保存用户选择的项目类型与配置
  • 域名管理
    • 子域名领取校验（检查是否已被占用）
    • 保存 subdomain → 项目映射表到数据库
  • 同步 KV
    • 当用户创建/修改小项目时，调用 Cloudflare API 将最新映射写入 KV
  • 后台管理
    • 查看所有用户项目
    • 统计访问量、热门项目
    • 可手动干预或修复 KV 映射
关键点： 服务器不直接响应用户访问请求，只处理写操作。KV 是 Worker 访问的“配置中心”，服务器负责保持其最新状态。

3️⃣ Cloudflare 端职责

  • DNS
    • 设置 *.example.com 通配符解析到 Worker
    • 用户访问的子域名无需单独添加记录
  • Worker
    • 拦截所有子域名请求
    • 解析 Host 获取子域名
    • 访问 KV 获取当前子域名对应的小项目类型和配置
    • 调用相应小项目模块生成页面
  • KV
    • 保存每个子域名的映射：subdomain → { projectType, configId }
    • Worker 读取 KV 生成页面
  • Edge Cache
    • 热门页面缓存，加速用户访问
    • 大幅减少 KV 和 Worker 执行次数
原则：边缘只做读操作、渲染页面和缓存，保证访问速度快、压力低。

4️⃣ 服务器与 CF 的通信

  1. 用户在平台创建或修改项目
  2. 服务器写数据库并校验合法性
  3. 服务器调用 Cloudflare KV API，将映射写入 KV
  4. Worker 读取 KV 渲染页面
  5. 用户访问子域名即可获取最新页面
核心逻辑：服务器写，Worker 读，服务器不参与每次访问。

5️⃣ 用户访问流程（技术视角）

  1. DNS 层
    • user.example.com 匹配通配符 *.example.com
    • 访问路由到 Worker
  2. Worker 层
    • 拦截请求
    • Host = user.example.com → 提取 subdomain = user
    • 查询 KV：KV.get("user") → { projectType: "abc", configId: 123 }
    • 调用对应模块 renderABC(123) 生成 HTML/JS 页面
  3. Edge Cache
    • 若缓存存在直接返回缓存页面
    • 否则生成页面并缓存
  4. 用户浏览器
    • 显示小项目页面
    • 页面内可触发 API 调用（例如点赞、留言），这些 API 也由服务器处理
  5. 用户切换项目
    • 平台操作更新数据库 + KV
    • 下次访问 user.example.com 自动显示新项目，无需 DNS 或 Worker 改动

6️⃣ 为什么这么做

问题 解决方案 原因
大量用户访问 Worker + KV + Cache 边缘处理，秒开页面，减轻服务器压力
用户切换项目 KV 存映射，Worker 动态调用 不改 DNS，不改 Worker，不重部署
子域名爆炸 通配符 DNS 不用为每个用户单独解析
管理复杂 服务器集中管理写操作 统一数据库，易扩展、易维护
高性能 缓存 + KV 读操作在边缘完成，延迟低

7️⃣ 技术亮点总结

  • 单 Worker 支撑所有小项目：模块按需调用、KV存储映射、Cache加速。
  • 服务器只写不读：高并发访问不会打到数据库。
  • 通配符 DNS + KV 映射：子域名动态绑定，即时生效。
  • 可扩展性强：边缘架构支撑数万子域名毫无压力。

✅ 总结

  • 服务器：用户管理 + 项目管理 + 数据持久化 + KV 同步
  • Cloudflare：边缘请求处理 + 子域名解析 + 小项目渲染 + 缓存
  • 连接方式：服务器写 KV → Worker 读 KV → Edge Cache 返回页面
